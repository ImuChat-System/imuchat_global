## 🔄 Workflow de Travail Recommandé

### Phase Planning

1. ✅ Review [ACTION_PLAN_PRIORITAIRE.md](../ACTION_PLAN_PRIORITAIRE.md) → TOP 10
2. ✅ Créer GitHub Issues / Notion tasks
3. ✅ Assigner priorités & responsables
4. ✅ Définir sprint (1-2 semaines)

### Phase Exécution

1. ✅ Daily : Cocher checklist avancement
2. ✅ Référence docs techniques selon besoin
3. ✅ Tests continus (cf. Checklist pré-lancement)

### Phase Review

1. ✅ Comparer réalisé vs. KPIs Phase 1
2. ✅ Feedback équipe + externes
3. ✅ Ajuster priorités Phase 2

## 💡 Tips Utilisation Documents

### ✅ DO

- Traiter documents comme référence vivante (update si évolution)
- Linker issues GitHub vers sections docs pertinentes
- Utiliser checklists comme task lists concrètes
- Partager extraits pertinents selon audience

### ❌ DON'T

- Ne pas dupliquer info entre docs (risque désynchronisation)
- Ne pas ignorer estimations temps (planning réaliste)
- Ne pas skipper checklist pré-lancement (qualité critique)
